# 手动 QA 清单 — Tab Manager 扩展

在 Chrome 中加载解压后的扩展进行验证。

## 1. 基本功能

- [ ] 侧面板打开：点击扩展图标 → 侧面板显示「标签管家」
- [ ] 标签总数：header 显示正确的标签数量
- [ ] 分组树：标签按规则引擎自动分组（开发 / 视频 / 购物 / 阅读……）
- [ ] 分组排序：标签数多的分组排在前面
- [ ] 展开/折叠：点击分组头部可折叠或展开标签列表
- [ ] 标签标题：每个标签显示标题和 favicon
- [ ] 点击标签：点击标签跳转到对应标签页

## 2. 搜索

- [ ] 搜索框：输入关键词实时过滤标签
- [ ] 清空搜索：点击 ✕ 按钮清空并恢复所有分组
- [ ] 搜索匹配：匹配标题和 URL 中的关键词
- [ ] 聚焦样式：输入框聚焦时出现蓝色边框

## 3. Tab 导航

- [ ] 分组 Tab：显示分组视图
- [ ] 建议关闭 Tab：显示 7 天以上未访问的标签
- [ ] 历史 Tab：显示最近关闭的标签历史
- [ ] Tab 切换：点击不同 tab 页签切换内容

## 4. 建议关闭

- [ ] 正确识别 7 天以上未访问的标签
- [ ] 排除 pinned、active、audible 的标签
- [ ] 单个关闭：点击「关闭」按钮关闭单个标签
- [ ] 全部关闭：点击「全部关闭」关闭所有建议标签
- [ ] 无建议时显示「✅ 没有需要关闭的标签」

## 5. 休眠功能

- [ ] 点击「😴 休眠不活跃标签」按钮
- [ ] 正确过滤 1 天以上未访问的标签
- [ ] 排除 pinned、active、audible 的标签
- [ ] 标签从当前窗口中消失（可恢复）

## 6. 规则匹配验证

- [ ] 打开 `github.com` → 归入「开发」分组
- [ ] 打开 `youtube.com` → 归入「视频」分组
- [ ] 打开 `amazon.com` → 归入「购物」分组
- [ ] 打开 PR 页面 → 归入「开发」（URL 模式匹配）
- [ ] 打开 `mail.google.com` → 归入「邮件」（URL 模式匹配）
- [ ] 新标签 `about:blank` → 归入「其他」
- [ ] 未知域名 → 归入「其他」

## 7. 历史记录

- [ ] 关闭标签后被记录在历史中
- [ ] 显示关闭时间
- [ ] 显示原始分组

## 8. 社区规则同步

- [ ] 扩展启动时自动拉取社区规则
- [ ] 社区规则拉取失败不影响已有功能
- [ ] 用户可以禁用社区规则同步

## 9. 异常场景

- [ ] 打开 50+ 标签时侧面板性能正常（无卡顿）
- [ ] 快速切换标签时分组自动更新
- [ ] 删除标签后对应分组更新
- [ ] Service Worker 休眠后重新唤醒功能正常

## 10. 打包前确认

- [ ] manifest.json 权限声明完整
- [ ] 图标文件（16/48/128）存在
- [ ] 最小 Chrome 版本声明
- [ ] 隐私政策 URL（如适用）
